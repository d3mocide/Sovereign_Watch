# INFRA-07: Cloudflare CDN Layer Removal

## Issue
The `infra-poller` was receiving persistent `403 Forbidden` responses when attempting to fetch Cloudflare CDN edge node locations. While bypasses were attempted using User-Agents, the proprietary endpoint proved unreliable for long-term automation and the data was deemed redundant with existing PeeringDB records.

## Solution
Instead of maintaining a fragile scraper for an unauthorized endpoint, the Cloudflare CDN infrastructure layer was entirely **removed** from the Sovereign Watch dashboard. This reduces noise, improves poller performance, and ensures only high-fidelity, authoritative data (like PeeringDB and DNS Root status) is displayed.

## Changes
- **Backend (`infra-poller/main.py`):**
    - Removed `cdn_edge_loop`, `_fetch_cdn_edge_nodes`, and `parse_cloudflare_locations`.
    - Removed `POLL_INTERVAL_CDN_HOURS` and associated constants.
    - Fixed an accidental regression in `parse_ioda_outages` caused during cleanup.
- **Frontend:**
    - Removed `CdnEdgeNode` types and associated map filters.
    - Purged CDN-specific UI fragments from `MapTooltip` and `InfraView`.
