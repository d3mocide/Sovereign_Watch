# Fix CDN Edge Node Fetch Error in Infra Poller

## Issue
The infra-poller was attempting to fetch Cloudflare CDN edge nodes from `speed.cloudflare.com/locations` but was receiving a `403 Forbidden` response. This was due to Cloudflare blocking the default `aiohttp` user agent.

## Solution
Added a standard browser `User-Agent` header to the `aiohttp.ClientSession` used in `_fetch_cdn_edge_nodes` to bypass the block.

## Changes
- `backend/ingestion/infra_poller/main.py`
  - In `_fetch_cdn_edge_nodes`, added `headers={"User-Agent": USER_AGENT}` to the `aiohttp.ClientSession` initialization.

## Verification
- Code successfully builds and lint passes.
- A standard browser `User-Agent` string has been confirmed working on Cloudflare previously (and used elsewhere in the same file).
