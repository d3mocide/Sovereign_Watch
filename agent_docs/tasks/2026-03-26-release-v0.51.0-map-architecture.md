# 2026-03-26-release-v0.51.0-map-architecture

## Issue
The map foundations and controls were duplicated across `TacticalMap`, `OrbitalMap`, and `IntelGlobe`, leading to architectural debt and inconsistent behavior. Additionally, high-frequency React state updates in the animation loop were impacting performance.

## Solution
- Extracted map base state and initialization into a reusable `useMapBase` hook.
- Consolidated map control UI into a shared `MapControls` component.
- Implemented an inline ref-sync pattern in `useAnimationLoop` and `IntelGlobe` to eliminate unnecessary React re-renders during animation.
- Centralized map style constants and lazy-loaded adapters in `mapStyles.ts`.

## Changes
### Frontend
- **`src/hooks/useMapBase.ts`**: New shared hook for map initialization.
- **`src/components/map/MapControls.tsx`**: New shared HUD control component.
- **`src/components/map/mapStyles.ts`**: New style and adapter constants.
- **`src/components/map/TacticalMap.tsx`**: Refactored to use shared foundations.
- **`src/components/map/OrbitalMap.tsx`**: Refactored to use shared foundations.
- **`src/components/map/IntelGlobe.tsx`**: Removed boilerplate and optimized animation loop.
- **`src/hooks/useAnimationLoop.ts`**: Added ref-sync support.
- **`src/utils/interpolation.ts`**: Added `baseAlpha` support.
- **`package.json`**: Bumped version to `0.51.0`.

## Verification

- **Visuals**: Confirmed map initialization and control parity across all mission views.

## Benefits
- Reduced codebase size (~400 lines removed from map components).
- Improved rendering performance (eliminated 60 re-renders/sec in Intel view).
- Standardized UX across Tactical, Orbital, and Intel modes.
